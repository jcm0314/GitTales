# [팀원 설득용 피칭 자료] GitTales (깃테일즈)
> "코딩테스트의 시대는 갔다. 이제는 과제테스트다." - 과제테스트용 백준 + AI 리뷰 튜터 

---

## 🖥️ 파트 1. 발표 슬라이드 구성 (Presentation Slides)

### 🛝 Slide 1. 오프닝 및 타이틀 (Title)
* **슬라이드 제목**: **GitTales (깃테일즈)**
* **부제목**: 백준처럼 풀고, AI에게 리뷰받고, 캐릭터를 키우는 실무 과제테스트 연습 & 1:1 피드백 플랫폼
* **핵심 흐름**: 
  1. 깃허브 실무 과제 제출 ➔ 2. AI 자동 코드 리뷰 및 채점 ➔ 3. 내 실수 기반 맞춤형 학습(퀴즈)

---

### 🛝 Slide 2. 문제 제기 (Problem)
* **슬라이드 제목**: 알고리즘 코딩테스트의 종말, 준비된 대안이 부재한 개발자 채용 시장
* **핵심 내용 (그리드 카드)**:
  * **코테 변별력 상실**: AI 자동완성(Copilot, ChatGPT)의 일상화로 알고리즘 코테는 변별력을 잃어 실제 코드를 구현하는 과제테스트 선호도 급증.
  * **피드백 차단 (주니어)**: 과제테스트 탈락 시 "내 코드가 왜 떨어졌고 어떤 부분이 부족한지" 알려주는 연습/피드백 플랫폼 전무.
  * **기업 및 교육기관의 막대한 채점 피로**: 수십 명의 복잡한 프로젝트 코드를 수동 채점해야 하여 시니어 개발자 리소스가 낭비되거나 부실한 피드백 전달.

---

### 🛝 Slide 3. 해결책 및 서비스 핵심 흐름 (Solution)
* **슬라이드 제목**: 실무 과제 제출부터 개인 취약점 극복까지, AI 기반의 선순환 훈련 루프
* **핵심 내용**:
  * **1단계: 실무 과제 기출문제 Clone**: 유저는 웹에서 제공하는 과제 보일러플레이트를 복사하여 개발 후 GitHub PR 제출.
  * **2단계: AI 자동 코드 리뷰 및 정량 채점**: 제출 즉시 AI가 코드 퀄리티 분석, 카테고리별 채점 및 깃허브 라인별 피드백 코멘트 실시간 작성.
  * **3단계: 실수 기반 RPG 게이미피케이션 학습**: 내 코멘트 기반 맞춤형 오답 퀴즈(객관식/Explain-Back)를 풀어 스텟(LTI 지수) 및 캐릭터 레벨업.

---

### 🛝 Slide 4. 차별점 및 해커톤 합격 요인 (Differentiation)
* **슬라이드 제목**: 시중 B2B 평가 도구들과 다른 GitTales만의 독보적 경쟁력
* **핵심 내용 (비교 로우)**:
  * **시중 AI 리뷰어 (B2B)**: 단순 정답 코드 즉시 제안 ➔ 개발자의 기계적 수용으로 생각하는 힘 상실 (학습 효과 X).
  * **시중 채용 툴 (B2B)**: 기업 전용 스크리닝 용도 ➔ 일반 취준생은 연습용 가입 및 피드백 획득이 원천 불가능.
  * **GitTales (B2C 오픈)**: 취준생 누구나 무료 실습 가능 + 내 실수를 개념 퀴즈로 가공해 학습을 완성시키는 에듀테크 지향.

---

### 🛝 Slide 5. 기술적 실현 가능성 (Technical Feasibility)
* **슬라이드 제목**: 무박 2일 내 핵심 데모(MVP) 완성을 위한 안정적인 초속성 개발 스택
* **핵심 내용 (기술 카드)**:
  * **동적 동기화 API 아키텍처**: 복잡한 깃허브 앱 심사 우회. GitHub REST API와 유저 개인 토큰(PAT) 연동으로 데모용 원클릭 수집.
  * **Gemini JSON Mode 엔진**: 코드 코멘트 데이터셋을 완벽한 스키마의 퀴즈 데이터(JSON)로 자동 파싱하여 백엔드 복잡도를 최소화.
  * **Supabase + Recharts RPG 대시보드**: Supabase Client-Side Direct Query로 초고속 DB 연동. Recharts로 개발자 성장을 상징하는 다이내믹 레이더(Radar) 차트 구현.

---

### 🛝 Slide 6. 역할 분담 및 마일스톤 (R&R & Roadmap)
* **슬라이드 제목**: 팀원 6인의 유기적 역할 분담 및 마일스톤
* **역할 매핑**:
  * **기획 및 디자인 (1명)**: RPG 비주얼 스킨 기획, 시연 영상 촬영/편집, 피칭 장표 제작.
  * **AI / Prompt 엔지니어 (1명)**: Gemini 퀴즈 생성 및 자기설명(Explain-Back) 검증 프롬프트 튜닝.
  * **풀스택 엔지니어 (1명)**: Supabase 세팅, GitHub REST API 수집 모듈 구현.
  * **프론트엔드 엔지니어 (2명)**: RPG 스탯 대시보드(Radar Chart) 및 인터랙티브 퀴즈 UI 구현.
  * **데이터 코디네이터 (1명)**: 테스트용 과제 Boilerplate 및 가상 리뷰 시나리오 설계.

## 🎙️ 파트 2. 발표 대본 (Word-by-word Script)

**(대본을 시작하기 전 가벼운 아이스브레이킹 제안: 팀원들의 공감을 먼저 자극하세요.)**

### 🎤 오프닝 (Slide 1)
> "여러분, 코딩테스트 준비할 때 백준이나 프로그래머스 많이 쓰시죠? 그런데 혹시 실제 채용에서 **'과제테스트'** 준비할 때는 어떤 사이트 쓰셨나요? 아마 마땅한 플랫폼이 없어서 깃허브에서 남의 코드 뒤적이거나 블로그 보고 맨땅에 헤딩하셨을 겁니다. 
> 
> 요즘 기업들은 알고리즘 코딩테스트보다 실제 프로젝트 구현 능력을 보는 과제테스트를 훨씬 선호하는 추세입니다. 하지만 이를 연습하고 피드백을 받을 수 있는 '과제테스트용 백준'은 시장에 전혀 없습니다. 
> 
> 그래서 우리가 이번 해커톤에서 만들 아이디어는, 백준처럼 실무 과제를 해결하고, AI에게 정교한 코드 리뷰를 받으며, 내가 실수한 내용을 기반으로 맞춤형 퀴즈를 풀어 레벨업하는 개발자 과제 훈련 플랫폼, **GitTales(깃테일즈)**입니다."

### 🎤 문제 제기 (Slide 2)
> "이 아이디어를 제안하는 명확한 이유가 있습니다. 첫째, 취준생 입장에서는 과제테스트에서 떨어져도 **'내 코드가 왜 틀렸고 어떤 부분이 부족한지'** 알려주는 곳이 전혀 없습니다. 
> 
> 둘째, 채용 기업이나 교육 기관(KDT 등) 입장에서는 수십 명의 과제 코드를 일일이 채점하고 고품질 피드백을 적어주는 데 엄청난 엔지니어 리소스와 비용이 듭니다. 
> 
> 셋째, GitHub Copilot 같은 AI 비서들이 코드를 대신 짜주는 시대에 주니어들의 **'스스로 생각하는 역량'**이 점점 저하되고 있습니다. 우리는 이 세 가지 문제를 한 번에 해결하고자 합니다."

### 🎤 서비스 흐름 (Slide 3)
> "사용자 시나리오는 매우 명확하고 재밌습니다. 유저가 우리 플랫폼에서 실무 과제 기출문제를 선택하고 제공된 보일러플레이트를 복사해 개발합니다. 다 짜고 깃허브에 Pull Request를 날린 후, 웹앱에서 **[리뷰 동기화]** 버튼을 누릅니다.
> 
> 그러면 AI가 5초 만에 코드를 분석해서 가독성, 성능, 보안 등의 관점으로 깃허브 PR에 직접 코드 리뷰 코멘트를 달아주고 점수를 매깁니다. 
> 
> 여기서 끝이 아닙니다. AI는 유저가 지적받은 문제 코드를 바탕으로 객관식 퀴즈와 빈칸 채우기 퀴즈를 출제합니다. 유저가 퀴즈를 풀고 스스로 개선 방향을 설명(Explain-Back)하면, 그 결과에 따라 **RPG 게임 캐릭터처럼 개발자 스텟(능력치 레이더 차트)과 레벨이 상승**합니다."

### 🎤 차별화 및 해커톤 우승 전략 (Slide 4)
> "기존에 AI가 코드를 대신 리뷰해서 수정본을 던져주는 CodeRabbit 같은 서비스는 주니어의 성장을 돕지 못합니다. 그냥 수동적으로 받아적기 때문이죠. 
> 
> 하지만 GitTales는 유저의 실수를 **교육용 퀴즈로 재가공**해서 지식을 머릿속에 락인(Lock-in)시킵니다. 
> 
> 특히 고용노동부가 주관하는 KDT 해커톤의 심사위원들은 단순한 유틸리티 툴보다 **'AI 시대에 인간 개발자의 역량을 어떻게 지속 가능하게 확장할 것인가'**라는 교육적 가치와 공공 성격에 크게 반응합니다. 이 관점에서 우리 아이디어는 예선 서면 심사는 물론 본선에서도 압도적인 정당성을 가집니다."

### 🎤 기술적 구현성 및 R&R (Slide 5 ~ 6)
> "가장 중요한 건 '이걸 무박 2일 동안 다 구현할 수 있느냐'입니다. 결론부터 말씀드리면 가능합니다.
> 
> 해커톤 기간 동안은 복잡한 깃허브 웹훅이나 프로덕션용 앱 심사 단계를 우회하고, 테스트용 레포지토리와 유저의 개인 토큰(PAT)을 활용하여 GitHub REST API로만 데이터를 동기화합니다. 
> 
> AI 영역은 Gemini API의 구조화된 JSON 출력을 사용하므로 프롬프트 엔지니어링만 정교하게 해두면 별도의 AI 백엔드 없이 Supabase를 통해 프론트엔드로 직접 데이터를 쏴줄 수 있습니다. UI는 Recharts와 Shadcn UI 같은 라이브러리를 활용해 캐릭터 스텟 차트를 아주 화려하게 구현할 예정이라 시각적 완성도도 보장됩니다.
> 
> 우리 팀원들의 역할을 백엔드 API 연동, Gemini 프롬프트 최적화, 프론트엔드 캐릭터 대시보드 개발, 테스트 시나리오 구축 등으로 역할을 명확히 나누면, 첫째 날 밤에 작동 가능한 핵심 기능을 완성하고, 둘째 날 오전에는 비디오 촬영과 발표 준비에 몰입할 수 있습니다. 
> 
> 알고리즘 중심의 뻔한 개발 도구가 아닌, 개발자 취업 시장의 판도를 바꿀 이 매력적인 플랫폼을 함께 만들어보면 좋겠습니다. 이상입니다. 감사합니다!"
